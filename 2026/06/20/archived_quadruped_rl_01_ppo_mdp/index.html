四足机器人强化学习系列 01：先把 MDP、Policy、PPO 讲明白
目录

先给结论

四足机器人里常用 PPO，不是因为它最“高级”，而是因为它在连续控制任务里足够稳定、工程实现成熟、并行采样效率高，适合让机器人在仿真中反复试错。

MDP 是什么

强化学习通常把问题抽象成 MDP：Markov Decision Process，马尔可夫决策过程。

在四足机器人里可以这样理解：

比如让机器人按命令速度向前走：

  • 状态：机身角速度、重力方向、关节角、关节速度、上一帧动作、速度命令。
  • 动作：12 个关节的位置目标或位置偏移。
  • 奖励：走得越接近目标速度越好，身体越稳越好，能耗越低越好。
  • 终止：摔倒、身体姿态过大、超时。

State 和 Observation 的区别

理论里常说 state，但机器人训练里经常用 observation。

  • state：完整真实状态，理论上包含系统全部信息。
  • observation：策略实际能看到的信息。

真实机器人不可能知道所有东西，比如精确地面摩擦、每个接触点真实接触力、完整环境模型。所以工程里更关心 observation。

Action 是什么

四足机器人动作通常不是“脚往前迈一步”这种高层语义，而是更低层的控制量。

常见设计：

  1. 关节位置目标：策略输出每个关节的目标角度，然后底层 PD 控制。
  2. 关节位置偏移：策略输出相对默认姿态的偏移量。
  3. 关节力矩：策略直接输出 torque，难度更高，安全风险更大。
  4. 残差动作：在传统控制器基础上加一个 RL 修正量。

很多四足 locomotion 训练会选择“位置目标/偏移 + PD”，因为稳定、易迁移、真机风险小一些。

Value 是什么

PPO 通常还会训练一个 value 网络。

Policy 负责回答：

我现在应该做什么动作？

Value 负责回答：

我现在这个状态大概有多好？未来能拿多少奖励？

它不是直接控制机器人，但能帮助算法判断某次动作是不是比预期更好。

为什么四足机器人常用 PPO

PPO 的核心思想是：每次更新策略不要走太猛，避免刚学会一点东西又被一次大更新毁掉。

这对四足机器人很重要，因为 locomotion 任务非常容易不稳定：

  • 一个动作不合适就会摔倒。
  • 奖励函数里很多项相互拉扯。
  • 接触动力学带来强非线性。
  • 仿真并行采样数据量很大，策略更新太激进会崩。

PPO 在四足训练里的流程

并行环境是四足 RL 的效率关键。比如 Isaac Gym 的思路是让大量仿真环境在 GPU 上并行跑，几分钟到几十分钟内收集大量经验。

四足机器人里 PPO 不是全部

学 PPO 是必要的，但不要误以为“会 PPO = 会四足强化学习”。真正影响效果的往往是：

  • observation 设计。
  • reward 设计。
  • action scale。
  • PD 参数。
  • terrain curriculum。
  • domain randomization。
  • reset 条件。
  • 训练命令分布。
