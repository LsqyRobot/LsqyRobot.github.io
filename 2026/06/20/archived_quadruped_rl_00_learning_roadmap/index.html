四足机器人强化学习系列 00：这套课应该怎么学
目录

写在前面：这套系列解决什么问题

我想系统学习四足机器人里的强化学习，但如果直接从强化学习教材开始，很容易遇到两个问题：

  1. 教材里的例子往往是 GridWorld、CartPole、Atari，和四足机器人距离比较远。
  2. 直接看四足论文和开源代码，又会被 PPO、reward、domain randomization、PD control、Sim2Real、privileged observation 等词淹没。

所以这套系列的目标不是把所有强化学习理论都讲完，而是围绕一个非常具体的方向：

如何理解、复现、修改四足机器人运动控制中的强化学习训练流程。

换句话说，我们关心的问题是：

  • 四足机器人强化学习到底在学什么？
  • 策略网络输入什么，输出什么？
  • 为什么常见算法是 PPO？
  • reward 为什么这么重要？
  • 仿真里训练出来的策略为什么不能直接上真机？
  • 如果我要读 legged_gym / rsl_rl / Isaac Lab 这类工程，应该先看哪里？
  • 如果我要自己做一个小实验，应该怎么安排？

这篇是第 00 篇，相当于课程路线图。

先给一句话结论

学习四足机器人强化学习，最合适的顺序不是先啃 PPO 公式，而是先建立“机器人观测—策略输出—仿真执行—奖励反馈—策略更新—真机迁移”的工程闭环。

也就是说，我们要先知道整个系统怎么转起来，再逐步深入每一块。

泛强化学习当然重要，但四足机器人有自己的特殊性。

因此如果只学通用 RL 理论，可能知道 policy gradient，但不知道 action scale 为什么会让机器人摔倒；知道 PPO clipping，但不知道 reward 里 torque penalty 太大会让机器人不愿意走；知道 domain randomization，但不知道真实机器人里 obs 顺序错了会直接炸。

四足 RL 的学习，必须同时有算法视角和工程视角。

这套课的文章长度标准

这里先明确一下后续写作风格。

我说的“短篇”，不是几百字的碎片，而是：

  • 单篇尽量控制在 1 万字以内。

总体路线图

我建议把四足机器人强化学习拆成 6 个阶段来学。

阶段 1：先知道四足 RL 在解决什么问题

这一阶段不急着看公式。

先建立一个基本图景：

在这个阶段，你需要能回答：

  • 四足机器人为什么可以用强化学习训练步态？
  • policy 是什么？
  • observation 是什么？
  • action 是什么？
  • reward 是什么？
  • done/reset 是什么？

如果这些问题不清楚，后面看 PPO 公式和工程代码都会很痛苦。

阶段 2：理解 Observation、Action、Reward

这是四足 RL 的核心三件套。

Observation

Observation 是机器人策略真正能看到的信息。

常见包括：

  • 机身角速度。
  • 重力方向在机身坐标系下的投影。
  • 速度命令。
  • 关节角。
  • 关节速度。
  • 上一帧 action。
  • 可能还有地形高度采样。

初学时要特别注意：obs 不是随便拼的，它必须和真机能获得的信息相匹配。

如果训练时用了真机上拿不到的信息，仿真效果可能很好，但部署时就会遇到问题。

Action

Action 是策略输出的控制量。

在很多四足 locomotion 工程里，action 不是直接输出电机力矩，而是输出关节位置目标偏移：

底层再用 PD 控制器转成力矩。

这种方式比直接输出 torque 更稳定，也更容易迁移到真实机器人。

Reward

Reward 是最值得认真学的部分。

它通常由很多项组成：

  • 速度跟踪奖励。
  • 角速度跟踪奖励。
  • 身体姿态惩罚。
  • 力矩惩罚。
  • 动作变化惩罚。
  • 脚滑惩罚。
  • 碰撞惩罚。
  • 足端空中时间奖励。

一句话：你奖励什么，机器人就会努力变成什么；你漏掉什么，机器人就可能钻空子。

阶段 3：再学 PPO

PPO 是四足机器人运动控制里非常常见的算法。

但我不建议一上来就陷入公式。

先记住 PPO 的核心直觉：

更新策略时不要一步走太猛。

四足机器人训练很容易不稳定。策略刚学会站住，如果一次更新太大，可能下一轮又全摔倒。

PPO 通过限制新旧策略差异，让训练更稳。

在工程里，你需要理解：

  • policy 网络输出 action。
  • value 网络估计当前状态好不好。
  • advantage 表示这一步比预期好多少。
  • rollout 是收集一段机器人交互数据。
  • update 是用这些数据更新 policy。

公式当然可以后面补，但第一阶段要先知道它在工程里扮演什么角色。

阶段 4：读懂训练工程

真正开始看代码时，不要从 PPO 文件开始。

建议先找这些位置：

  1. 机器人模型在哪里加载。
  2. observation 在哪里拼接。
  3. action 怎么变成关节目标。
  4. reward 在哪里计算。
  5. reset 条件在哪里。
  6. command 怎么采样。
  7. domain randomization 在哪里配置。
  8. 日志保存在哪里。

这比直接看算法更新更重要。

因为同一个 PPO，在不同 obs、reward、action 设计下，结果会完全不同。

常与 legged_gym 配合使用的强化学习库，里面有 PPO runner、actor critic 等实现。

NVIDIA 的 GPU 并行仿真训练生态。四足 RL 之所以能快速训练，很大程度依赖大规模并行仿真。

仓库：

https://github.com/isaac-sim/IsaacLab

学习时最容易踩的坑

坑 1：只看 PPO，不看 reward

PPO 很重要，但很多四足训练效果差，不是 PPO 错了，而是 reward、obs、action scale、PD 参数、reset 条件设计有问题。

如果只改算法，不看环境，容易抓错重点。

坑 2：看视频觉得会走了，就以为训练好了

会走不代表好。

还要看：

  • 速度跟踪误差。
  • 能耗。
  • 动作是否抖。
  • 足端是否打滑。
  • 对扰动是否鲁棒。
  • 不同地形是否稳定。
  • 长时间是否会漂。

坑 3：仿真能走，就想直接上真机

真机部署风险很高。

仿真和现实之间有很多差异：

  • 质量和惯量不准。
  • 摩擦系数不准。
  • 电机延迟。
  • IMU 噪声。
  • 关节零位误差。
  • 控制频率不一致。
  • action scale 或 obs 顺序写错。

Sim2Real 不是最后一步随便做一下，而是从训练设计阶段就要考虑。

坑 4：不知道日志怎么看

训练时不要只看 total reward。

建议至少记录：

  • episode length。
  • tracking reward。
  • torque penalty。
  • action rate penalty。
  • reset rate。
  • velocity tracking error。

否则训练坏了很难定位原因。

推荐学习节奏

如果时间比较充裕，可以按下面方式学。

第 1 周：建立概念

读 00～05。

目标：知道 obs、action、reward、done 是什么。

练习：用自己的话画出四足 RL 闭环。

第 2 周：理解 PPO 和训练流程

读 08～10。

目标：知道 PPO 在工程里做什么，知道训练脚本大概怎么跑。

练习：找一个开源工程，定位 obs、reward、action mapping。

第 3 周：重点学 reward

反复读 05～07，并开始看代码里的 reward term。

目标：知道每个 reward 项会影响什么行为。

练习：改一个 reward 权重，观察训练结果变化。

第 4 周：理解 Sim2Real

读 11～13。

目标：知道仿真策略为什么不能直接上真机。

练习：写一份真机部署前检查表。

第 5～8 周：复现实验

按 14 的练习路线，跑通一个小任务。

目标不是马上做出很强的策略，而是建立完整流程：

学到什么程度算入门

如果你能做到下面这些，就算真正入门了：

  1. 能解释四足 RL 的训练闭环。
  2. 能说清 obs 里常见量的作用。
  3. 能说清 action 为什么常用关节位置目标。
  4. 能读懂 reward 的主要项。
  5. 能跑通一个开源训练工程。
  6. 能看训练曲线判断是否变好。
  7. 能做一个小的 reward 消融实验。
  8. 知道 Sim2Real 的主要风险。

这比单纯背 PPO 公式更有用。
