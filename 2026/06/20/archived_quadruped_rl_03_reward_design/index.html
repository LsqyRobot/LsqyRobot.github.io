先给结论

四足机器人强化学习里，奖励函数不是“打分表”这么简单，它更像行为塑形器：你奖励什么，机器人就会努力变成什么；你漏掉什么，它就可能钻空子。

姿态稳定奖励

机器人不能只顾往前冲，还要身体稳定。

常见约束：

  • roll / pitch 不要太大。
  • base height 不要太低。
  • projected gravity 接近正常方向。
  • 角速度不要过大。

如果姿态惩罚太弱，机器人可能出现低头、趴地、扭动身体等奇怪策略。

能耗与力矩惩罚

力矩惩罚通常写成：

它会鼓励机器人省力。

但注意：

  • 惩罚太小：动作可能暴力、费电、真机风险高。
  • 惩罚太大：机器人可能不愿意动，或者步态很保守。

  • IMU 噪声放大。

脚端相关奖励

四足步态离不开接触。常见设计包括：

  • 脚滑惩罚：接触地面时脚端水平速度不要太大。
  • 空中时间奖励：鼓励形成自然摆腿。
  • 碰撞惩罚：小腿/大腿不要撞地。
  • 足端高度奖励：摆腿时抬脚，避免绊倒。

脚端奖励很容易影响步态风格。

例如：

  • 空中时间奖励太强，可能出现夸张抬腿。
  • 脚滑惩罚太弱，机器人可能拖脚。
  • 足端高度过强，可能动作不自然。

为什么 reward 会被钻空子

强化学习只优化你写出来的目标，不优化你脑子里的目标。

如果你写：

机器人可能学会：

  • 摔倒后滑行。
  • 跳一下往前蹭。
  • 身体扭动产生位移。

所以要配合：

  • 姿态约束。
  • 高度约束。
  • 碰撞惩罚。
  • reset 条件。
  • 能耗惩罚。

奖励调参建议

我建议初学者按这个顺序调：

  1. 先保证机器人能站住。
  2. 再加入低速前进。
  3. 再加入角速度命令。
  4. 再加入更大速度范围。
  5. 再加入地形或扰动。
  6. 最后优化能耗和步态美观。

不要一开始就把所有奖励项拉满。

总结

奖励函数决定了机器人“想成为什么”。四足机器人 RL 里，奖励设计往往比 PPO 公式更影响最终效果。
