四足机器人强化学习系列 05：给自己的复现实验清单
目录

目标

这一篇不是讲新理论，而是给一个可执行的练习路线。目标是：从看懂文章，逐步走到能改一个四足 RL 训练任务。

第 1 周：读懂最小概念

你需要能解释：

  • observation 是什么。
  • action 是什么。
  • reward 是什么。
  • episode 是什么。
  • policy 和 value 的区别。
  • PPO 为什么要限制更新幅度。

练习：

  1. 写一个 CartPole 或 Pendulum 的 PPO 训练笔记。
  2. 把 obs/action/reward/done 画成流程图。
  3. 不看公式，用自己的话解释 advantage。

第 2 周：跑通现成四足仿真

目标：跑通一个公开工程，而不是马上改代码。

建议检查项：

  • 环境能创建。
  • 训练能开始。
  • 日志能记录。
  • play 能加载训练结果。
  • 能保存 policy。

记录这些信息：

第 3 周：定位关键代码

在工程里找到：

  • obs 计算函数。
  • reward 计算函数。
  • reset 判断函数。
  • action 到 torque/position target 的映射。
  • command 采样函数。
  • domain randomization 配置。

练习：给每个函数写中文注释，不需要改逻辑。

第 4 周：改一个 reward

选择一个最小改动：

  • 增大 action rate penalty。
  • 增大 torque penalty。
  • 降低目标速度范围。
  • 增加 base height 惩罚。

然后对比：

  • 训练曲线变化。
  • 机器人动作变化。
  • reward term 变化。
  • 是否更稳或更保守。

第 5 周：做一次消融实验

例如只改 torque penalty：

记录：

  • 速度跟踪误差。
  • 平均 torque。
  • episode length。
  • 视频现象。

这一步会让你真正理解奖励权重不是玄学，而是可以实验比较的工程参数。

第 6 周：加入扰动或随机化

尝试：

  • 随机摩擦。
  • 随机质量。
  • 加外力 push。
  • 加观测噪声。

观察策略是否更鲁棒。

第 7 周：整理 Sim2Real 检查表

即使没有真机，也可以先写检查表：

  • obs 顺序。
  • action 顺序。
  • action scale。
  • PD 参数。
  • 控制频率。
  • 关节限位。
  • torque limit。
  • 安全停机条件。
  • 日志字段。

第 8 周：形成自己的模板

最终你应该有一个自己的四足 RL 学习模板：

推荐长期方向

学完基础 locomotion 后，可以继续深入：

  • rough terrain locomotion。
  • blind locomotion vs vision-based locomotion。
  • teacher-student / privileged learning。
  • RMA 自适应。
  • imitation learning。
  • whole-body control + RL。
  • wheel-legged robot RL。
  • real-world deployment safety。
